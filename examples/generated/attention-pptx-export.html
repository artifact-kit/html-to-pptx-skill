1 Attention 计算结构：从嵌入到上下文向量

三类投影

Query 负责发起检索，Key 负责被匹配，Value 承载最终被汇聚的信息。

分布化选择

相似度分数经过缩放、mask 和 softmax，变成每个 token 对其它 token 的注意力分布。

解释：每一行表示一个 query 如何从所有 value 中取信息；颜色更深代表信息汇聚权重更高。
2 Multi-Head Attention
多个 head 并行读取不同关系：局部邻近、长程依赖、句法角色或指代线索。
3 Mask 与归一化

Padding mask: 屏蔽无效 token。

Causal mask: 自回归任务只允许访问过去位置。

保留下三角区域，屏蔽当前位置之后的 token。

4 Example Attention Matrix

深色代表更强的连接；局部块状结构常对应短语、句法或共指关系。

5 Complexity Scaling

标准 attention 需要比较所有 token 对；长上下文下，连接数量快速增长。

结构词汇

Query：发起检索 · Key：提供匹配索引 · Value：提供被读取的信息 · Head：一个并行关系视角

直觉

Attention 是可微分的检索机制：每个 token 先找到相关 token，再从这些位置读取信息，形成上下文化表示。

稳定性

缩放避免分数过大，mask 控制可见范围，残差和 LayerNorm 保持深层网络的训练稳定。
